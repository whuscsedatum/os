二、第1章 操作系统绪论
============================================================

1. 操作系统的地位、作用、功能
   【录音依据】00:01 老师说前面 1.1 到 1.5 基本是基本概念，要理解操作系统的地位、作用、功能。
   【知识点】
   - 操作系统位于硬件和应用程序之间。
   - 操作系统既是用户与计算机硬件之间的接口，也是计算机系统资源的管理者。
   - 操作系统的功能包括处理机管理、存储器管理、设备管理、文件管理、用户接口等。
   【考试要求】
   - 主要是选择/判断/简答层面的基础概念。
   - 不一定是大题核心，但概念要清楚。

2. 操作系统的三类接口
   【录音依据】00:01 老师说“三种接口”要知道，模糊转写中“图形的接口、……系统调用”应理解为：命令接口、程序接口、图形接口。
   【知识点】
   - 命令接口：用户通过命令与操作系统交互。
   - 程序接口：程序通过系统调用请求操作系统服务。
   - 图形接口：通过图形化界面与系统交互。
   【考试要求】
   - 要会区分三类接口。
   - 特别注意：系统调用属于程序接口，不是命令接口。

3. 单道程序与多道程序
   【录音依据】00:01 老师说要理解多道程序和单道程序相比带来的好处，并且曾经画图计算 CPU 利用情况。
   【知识点】
   - 单道程序：一个程序独占系统资源，遇到 I/O 等待时 CPU 可能空闲。
   - 多道程序：多个程序同时驻留内存，一个程序等待 I/O 时，CPU 可以执行另一个程序。
   - 直接效果：提高 CPU 利用率，提高系统吞吐量。
   【考试要求】
   - 可能给图或时间片，让你比较单道/多道下 CPU 利用率。
   - 重点是理解“多道程序通过并发提高资源利用率”。

三、第2章 相关硬件基础
============================================================

1. CPU 的工作方式
   【录音依据】01:05 老师说硬件基础是普通内容，重点理解 CPU 工作方式。
   【知识点】
   - CPU 的基本工作流程：取指、译码、取操作数、执行指令。
   - CPU 执行程序时，按指令周期不断推进。
   【考试要求】
   - 主要是基础概念。

2. 中断在操作系统中的作用
   【录音依据】01:05 老师说要理解中断，以及中断在操作系统中的重要作用。
   【知识点】
   - 中断是操作系统获得 CPU 控制权、响应外部事件和异常情况的重要机制。
   - 操作系统很多功能依赖中断机制，例如 I/O 完成通知、时钟中断、异常处理、系统调用入口等。
   【考试要求】
   - 要知道中断为什么对操作系统重要。
   - 不只是“打断程序”，而是 CPU 从用户程序转向操作系统内核处理事件的机制。

3. 中断、异常、系统调用
   【录音依据】01:05 老师说“中断的分类”里要看中断、异常和系统调用是什么意思。
   【知识点】
   - 中断：通常指来自 CPU 外部的异步事件，如 I/O 设备完成。
   - 异常：通常指程序执行过程中 CPU 内部产生的问题，如除零、越界、缺页等。
   - 系统调用：用户程序主动请求操作系统服务，是用户态进入内核态的一种方式。
   【考试要求】
   - 要能区分三者来源和性质。
   - 系统调用经常和用户态/内核态切换联系起来考。

四、第3章 进程
============================================================

1. 进程章节开始进入重点
   【录音依据】01:57 老师说第三章进程开始进入重点部分。
   【知识点】
   - 进程是操作系统中描述并发执行程序的核心概念。
   - 后续进程控制、进程状态、PCB、上下文切换都属于重要内容。

2. 进程控制代码题
   【录音依据】01:57 老师说大题点是进程控制里面的代码题，要能读懂代码，看输出、进程关系、进程框架关系。
   【知识点】
   - fork：创建子进程。
   - exec/execl：用新程序替换当前进程的程序映像。
   - wait/waitpid：父进程等待子进程结束。
   - getpid/getppid：获取进程号/父进程号。
   【考试要求】
   - 会读 fork/exec/wait/getpid 相关代码。
   - 会判断共产生多少个进程。
   - 会判断哪些语句由哪些进程执行。
   - 会画父子进程关系图。
   - 会分析输出可能性：不同进程并发执行，输出顺序可能不唯一。

3. fork 后父子进程的地址空间和变量
   【录音依据】02:40 老师说要理解进程是怎么通过 fork 复制出一模一样的地址空间，以及哪些资源共享、变量怎么修改。
   【知识点】
   - fork 后子进程得到父进程地址空间的副本。
   - 父子进程执行路径从 fork 返回后开始分叉。
   - 父进程中 fork 返回子进程 PID，子进程中 fork 返回 0。
   - 普通变量在父子进程中通常各自修改各自的副本，不是同一个变量。
   【考试要求】
   - 代码题中要分清父进程和子进程各自执行了什么。
   - 不要把父子进程中的局部变量/普通全局变量默认当成同一个。

4. 进程与程序的区别
   【录音依据】02:40 老师说前面的基本概念包括进程和程序的区别。
   【知识点】
   - 程序是静态的代码和数据集合。
   - 进程是程序的一次执行过程，是动态实体。
   - 同一个程序可以对应多个进程。
   【考试要求】
   - 基础概念题常考。

5. 进程映像
   【录音依据】02:40 老师点名“进程镜像是什么东西”。
   【知识点】
   - 进程映像/进程映像通常包括程序代码、数据、栈、堆、PCB 等与进程执行有关的内容。
   【考试要求】
   - 知道它描述的是一个进程在内存和系统中的完整存在形态。

6. PCB（进程控制块）
   【录音依据】02:40 老师点名“进程控制块是什么东西”。
   【知识点】
   - PCB 是操作系统描述和管理进程的数据结构。
   - PCB 保存进程标识、状态、程序计数器、寄存器现场、调度信息、资源信息等。
   【考试要求】
   - 要知道 PCB 是操作系统管理进程的核心数据结构。
   - 进程切换时，现场信息要保存/恢复到 PCB 相关结构中。

7. 进程上下文切换
   【录音依据】02:40 老师点名“进程的上下文切换是什么意思”。
   【知识点】
   - 上下文切换：CPU 从一个进程切换到另一个进程时，保存当前进程现场，恢复另一个进程现场。
   - 包括寄存器、程序计数器、栈指针等执行现场。
   【考试要求】
   - 可能与系统调用、中断、调度结合考。

8. 进程三态、五态、七态与挂起
   【录音依据】02:40 老师说三态、五态、七态都要理解一点，七态不一定要画，但要知道挂起状态为什么引入。
   【知识点】
   - 三态：运行、就绪、阻塞。
   - 五态：通常加入新建、终止。
   - 七态：进一步加入挂起相关状态，如就绪挂起、阻塞挂起。
   - 挂起：进程暂时被调出内存或暂停参与调度，用于缓解内存压力、系统管理或用户控制等。
   【考试要求】
   - 不一定要求完整画七态图。
   - 但必须知道为什么引入挂起状态。

9. 进程通信
   【录音依据】03:22 老师说进程通信简单看一下，不用作为大题重点。
   【知识点】
   - 进程通信方式可包括共享存储、消息传递、管道等。
   【考试要求】
   - 简单理解即可，不按老师录音看作重点大题。

五、第4章 线程
============================================================

1. 线程代码题
   【录音依据】03:41 老师说线程创建相关函数每年会和进程创建一起考大题，可能 10 分甚至 15 分。
   【知识点】
   - pthread_create：创建线程。
   - pthread_join：等待线程结束。
   - 线程函数、参数传递、返回值等要能看懂。
   【考试要求】
   - 会读线程创建代码。
   - 会判断创建了几个线程。
   - 会判断线程之间共享什么、输出可能是什么。
   - 线程代码题和进程代码题是同一类重点题。

2. 线程引入的原因
   【录音依据】03:41 老师说要理解线程引入之后实现了什么。
   【知识点】
   - 早期进程有两个属性：资源拥有的独立单位；调度和分派的基本单位。
   - 引入线程后：进程仍然是资源拥有单位；线程成为 CPU 调度和分派的基本单位。
   - 线程的目的：减少并发执行的时空开销，提高并发性。
   【考试要求】
   - 要能说清“线程不是为了取代进程，而是把调度单位从进程中拆出来”。

3. 进程与线程的区别
   【录音依据】03:41 老师提到线程和进程的区别、资源利用、实现内部体现。
   【知识点】
   - 进程拥有独立地址空间和资源。
   - 同一进程内线程共享代码段、数据段和操作系统资源。
   - 每个线程有自己的线程 ID、程序计数器、寄存器集合、栈等执行现场。
   【考试要求】
   - 重点是“资源归进程，调度归线程”。
   - 代码题中尤其要注意：线程通常共享全局变量，而 fork 后的进程通常不是共享普通变量。

4. 线程实现、线程库、操作系统例子
   【录音依据】04:24 附近老师提到 4.1、4.4 等位置；04:38 部分听不清，但能确定线程相关部分要看。
   【知识点】
   - 线程定义。
   - 线程实现。
   - 线程库。
   - 操作系统中的线程例子。
   【考试要求】
   - 没有明确说这一块出大题，但与线程代码题有关的内容要掌握。

六、第5章 进程调度
============================================================

1. 第5章重点是调度算法
   【录音依据】04:47 老师说第五章重点就是算法，重点就是调度算法。
   【知识点】
   - FCFS：先来先服务。
   - SJF：短作业优先。
   - SRTF：最短剩余时间优先。
   - 优先级调度。
   - 时间片轮转 RR。
   - 多级队列。
   - 多级反馈队列。
   【考试要求】
   - 会根据进程到达时间、运行时间、优先级等画调度过程。
   - 会算评价指标。

2. 多级反馈队列
   【录音依据】04:47 老师明确说掌握到多级反馈队列，课后题也布置过。
   【知识点】
   - 多个就绪队列，不同队列优先级不同。
   - 高优先级队列可抢占低优先级队列。
   - 不同队列可使用不同时间片。
   - 进程可能因用完时间片而降级，也可能因等待太久而提升优先级。
   【考试要求】
   - 往年大题可能要求设计或执行改进多级反馈队列调度。
   - 要写清楚队列规则、抢占规则、时间片规则、升级/降级规则。

3. 调度评价指标
   【录音依据】05:18 老师说要掌握调度算法的评估指标，会算周转时间等。
   【知识点】
   - 完成时间。
   - 周转时间 = 完成时间 - 到达时间。
   - 等待时间 = 周转时间 - 运行时间。
   - 带权周转时间 = 周转时间 / 运行时间。
   - 平均周转时间、平均等待时间、平均带权周转时间。
   【考试要求】
   - 计算题中必须列出调度顺序和各指标。
   - 只写结果容易因过程不清丢分。

4. 调度层次
   【录音依据】05:18 老师说要知道调度的层次，高级调度、中级调度、低级调度。
   【知识点】
   - 高级调度：作业调度，决定哪些作业进入内存/系统。
   - 中级调度：交换调度，决定哪些进程换入/换出。
   - 低级调度：进程调度，决定 CPU 分给哪个就绪进程。
   【考试要求】
   - 要能把三种调度和“外存—内存—CPU”层次对应起来。

5. 实时调度相关内容

   【处理】
   - 本文件不把实时调度列为老师明确强调的大题重点。
   - 只建议按 PPT 简单了解硬实时/软实时、中断延迟、分派延迟等。

   【录音依据】06:24 老师说写法一定要规范：定义信号量、初值、用途、伪代码、进程。
   【知识点】

   【录音依据】07:01 老师说能分析出一部分互斥关系或同步关系就会给部分分。

   【考试要求】
   - 做 PV 题第一步不是直接写 P/V，而是先分析关系。

1. 第7章常考大题

   - 请求到来时，将大块不断二分直到满足合适大小。

   【考试要求】

5. 分页、分段、段页式
   【录音依据】11:34 老师说分页、分段和段页式里面重点是分页，也不排除考分段和段页式。
   【知识点】
   - 分页：逻辑空间分成页，物理内存分成页框，页表记录页到块的映射。
   - 分段：按程序逻辑结构划分段，每段有段号和段内地址。
   - 段页式：先分段，每段再分页。
   【考试要求】

   - 分段和段页式也要会基本地址变换。

6. 分页地址变换
   【录音依据】11:34 老师说分页要知道地址变换。
   【知识点】
   - 逻辑地址分为页号和页内位移。
   - 页号查页表得到物理块号。
   - 物理地址 = 物理块号 + 页内位移。
   【考试要求】
   - 会根据页面大小拆分地址。
   - 会查页表并得到物理地址。

7. EAT 有效访问时间与快表 TLB
   【录音依据】11:34 老师说要会算 EAT，特别是有快表之后的 EAT。
   【知识点】
   - EAT：Effective Access Time，有效访问时间。
   - TLB/快表命中时减少页表访问时间。
   - TLB 未命中时通常需要访问页表再访问内存。
   【考试要求】
   - 会根据命中率、内存访问时间、快表访问时间计算平均访问时间。

8. 分段地址变换
   【录音依据】11:34 老师说分段也要知道地址变换怎么做。
   【知识点】
   - 逻辑地址包括段号和段内位移。
   - 查段表得到段基址和段长。
   - 若位移越界则产生越界中断。
   - 不越界时，物理地址 = 段基址 + 段内位移。
   【考试要求】
   - 要进行越界判断。

9. 段页式地址变换
   【录音依据】11:34、13:58 老师说段页式要理解先分段再分页；算段号、页号，找段表项，再找页表项，最后找到数据。

   - 段表项指向该段对应的页表。
   - 再用页号查页表，得到物理块号。
   - 最后结合页内位移得到物理地址或访问数据。
   【考试要求】

   【录音依据】12:24 老师说重点掌握请求分页存储系统里的地址变换，以及它和基本分页的区别。
   【知识点】

   - 页表项中有存在位/有效位等信息。
   - 如果页面在内存中，直接完成地址变换。

3. 页面置换算法
   【录音依据】12:24 老师说页面置换算法经常考大题，要根据访问序列算缺页和置换过程。
   【知识点】
   - FIFO：先进先出。
   - LRU：最近最久未使用。
   - Clock：时钟置换算法。
   - 可能还包括 OPT 等基本算法。
   【考试要求】
   - 会根据页面访问串和物理块数，逐步列出内存页框变化。
   - 会计算缺页次数和缺页率。
   - 要写过程，不要只给最后答案。

4. 页面分配策略与置换范围
   【录音依据】12:24 老师说固定分配、可变分配、全局置换、局部置换这些概念肯定要知道。
   【知识点】
   - 固定分配：给进程分配固定数量页框。
   - 可变分配：进程页框数可动态变化。
   - 全局置换：缺页时可从全系统范围选择被换出页。
   - 局部置换：缺页时只在本进程已分配页框中置换。
   【考试要求】
   - 会区分概念。
   - 可能考不能组合/可以组合的问题。

5. 工作集模型
   【录音依据】13:18 老师说要看工作集模型的作用和意义，以及它怎样通过局部性原理降低缺页率、避免抖动。
   【知识点】
   - 工作集：某段时间窗口内进程实际访问的页面集合。
   - 工作集模型利用程序局部性估计进程当前真正需要的页面。
   - 如果给进程足够容纳工作集的页框，可以减少缺页。
   【考试要求】
   - 要理解工作集不是随便分页框，而是根据近期访问行为估计局部性。

6. 抖动
   【录音依据】13:18 老师说抖动的概念是什么、影响因素要看。
   【知识点】
   - 抖动：系统频繁发生缺页和页面置换，大量时间用于换页，实际执行进程的时间很少。
   - 原因通常与内存不足、进程数过多、工作集得不到满足有关。
   【考试要求】
   - 要知道抖动与缺页率、工作集、多道程序度有关。
   - 优化方向可能是减少多道程序度、增加页框、改善置换策略等。

7. 影响缺页率的因素
   【录音依据】13:18 老师说影响缺页率的因素要大概看一下。
   【知识点】
   - 分配给进程的页框数。
   - 页面大小。
   - 页面置换算法。
   - 程序局部性。
   - 页面分配和置换策略。
   【考试要求】
   - 主要是概念理解题。
